# Ключевые проверки формы/документа на основе XSD-схемы

## Контекст
XSD-схема утверждена Приказом ФНС России № ЕД-7-26/383@ от 14.05.2024 (версия 5.01, КНД 1110442 для квитанции о приёме груза). Схема задаёт структуру XML-файла обмена информацией.

Ниже перечислены проверки, вытекающие из структуры XSD, включая обязательность полей, ограничения длины, форматы данных, schematron-правила.

---

## 1. Структура файла обмена (корневой элемент «Файл»)

| № | Проверка | Основание (XSD) | Тип |
|----|----------|-----------------|-----|
| F-01 | Атрибут `ИдФайл` обязателен, длина 1–255 символов | `@ИдФайл use="required"` | Обязательное поле |
| F-02 | Имя файла должно совпадать со значением `ИдФайл` | Schematron: `usch:getFileName() = @ИдФайл` | Schematron-правило |
| F-03 | `ВерсПрог` обязателен, длина 1–40 символов | `@ВерсПрог use="required"` | Обязательное поле |
| F-04 | `ВерсФорм` обязателен, фиксированное значение «5.01» | `@ВерсФорм use="required"` + enum | Обязательное поле |
| F-05 | `ИдПолИной` — опциональный, может повторяться (maxOccurs="unbounded"), длина 4–46 | `minLength=4 maxLength=46` | Опциональное |

## Примечания

1. **Для транспортной накладной (не квитанции) структура аналогична**, но с другими названиями элементов (например, `СодИнфПрв` → `СодИнфГО` и т.д.) и КНД = 1110440.
2. Все поля типа `decimal` должны проходить валидацию на соответствие формату (totalDigits, fractionDigits).
3. Schematron-правила — это дополнительные бизнес-правила поверх XSD, встроенные в файл через аннотации `xs:appinfo`.
4. Для тестирования API приёма документа критично: при нарушении XSD должен возвращаться HTTP 422 с детализацией ошибки (путь к элементу, конкретное правило).
